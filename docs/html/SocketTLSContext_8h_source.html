947/* ============================================================================
948 * Certificate Pinning (SPKI SHA256)
949 * ============================================================================
950 *
951 * OWASP-recommended approach: Pin the Subject Public Key Info (SPKI) hash.
952 * SPKI pinning survives certificate renewal when the same key is reused,
953 * making it more maintainable than full certificate pinning.
954 *
955 * Usage:
956 * // Create client context
957 * SocketTLSContext_T ctx = SocketTLSContext_new_client("ca-bundle.pem");
958 *
959 * // Pin by hex-encoded SHA256 hash
960 * SocketTLSContext_add_pin_hex(ctx,
961 * "b5bb9d8014a0f9b1d61e21e796d78dccdf1352f23cd32812f4850b878ae4944c");
962 *
963 * // Or extract from certificate file
964 * SocketTLSContext_add_pin_from_cert(ctx, "server.crt");
965 *
966 * // Enable strict enforcement (default, fail if no match)
967 * SocketTLSContext_set_pin_enforcement(ctx, 1);
968 *
969 * Generate pin from certificate:
970 * openssl x509 -in cert.pem -pubkey -noout | \
971 * openssl pkey -pubin -outform DER | \
972 * openssl dgst -sha256 -hex
973 *
974 * Thread safety: Pin configuration is NOT thread-safe. Configure all pins
975 * before sharing the context across threads. Verification is read-only
976 * and thread-safe.
977 */
978

1198/* ============================================================================
1199 * Certificate Transparency (RFC 6962)
1200 * ============================================================================
1201 *
1202 * CT helps detect mis-issued certificates by requiring them to be logged
1203 * in publicly auditable CT logs. Clients verify Signed Certificate
1204 * Timestamps (SCTs) embedded in certificates, TLS extensions, or OCSP.

1206 * Requires OpenSSL 1.1.0+ with CT support compiled in.

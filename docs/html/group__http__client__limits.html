◆ HTTPCLIENT_MAX_AUTH_RETRIES

#define HTTPCLIENT_MAX_AUTH_RETRIES   2

Maximum retries for HTTP authentication challenges (2).

Limits automatic retries on 401 Unauthorized responses when credentials are provided. Prevents infinite loops from servers repeatedly challenging without accepting valid creds. 2 allows for nonce refresh in Digest auth or transient server issues.

Applies to Basic (preemptive), Digest (challenge-response), and Bearer (token validation). Exceeding limit raises SocketHTTPClient_ProtocolError or falls back to 401 response. Not configurable at runtime; compile-time only for security.

See also
SocketHTTPClient_Auth for configuring credentials.
HTTP_AUTH_DIGEST handling of stale nonces.
SocketHTTPClient_Config no direct field; fixed limit.

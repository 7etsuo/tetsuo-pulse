243/* ============================================================================
244 * Authentication Types
245 * ============================================================================
246 *
247 * Supported authentication schemes:
248 *
249 * HTTP_AUTH_BASIC (RFC 7617):
250 * - Simple base64(username:password) encoding
251 * - Credentials sent with every request (no challenge-response)
252 * - Only use over HTTPS (credentials transmitted in cleartext)
253 *
254 * HTTP_AUTH_DIGEST (RFC 7616):
255 * - Challenge-response authentication with MD5 or SHA-256
256 * - Supports qop=auth (integrity protection of request)
257 * - Automatic 401 retry with proper response generation
258 * - Handles stale nonce refresh automatically
259 * - NOTE: qop=auth-int (body integrity) is NOT supported
260 *
261 * HTTP_AUTH_BEARER (RFC 6750):
262 * - OAuth 2.0 bearer token authentication
263 * - Token sent in Authorization header
264 * - Application responsible for token management/refresh
265 *
266 * NOT SUPPORTED:
267 * - NTLM (Microsoft proprietary, requires DES/MD4/NTLMv2)
268 * - Negotiate/SPNEGO (requires GSSAPI/Kerberos integration)
269 * - AWS Signature (v2/v4) - use dedicated AWS SDK
270 * - Hawk, HOBA, Mutual, OAuth 1.0a
271 * ============================================================================ */
272

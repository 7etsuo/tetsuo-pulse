362/* ============================================================================
363 * Authentication Types
364 * ============================================================================
365 *
366 * Supported authentication schemes:
367 *
368 * HTTP_AUTH_BASIC (RFC 7617):
369 * - Simple base64(username:password) encoding
370 * - Credentials sent with every request (no challenge-response)
371 * - Only use over HTTPS (credentials transmitted in cleartext)
372 *
373 * HTTP_AUTH_DIGEST (RFC 7616):
374 * - Challenge-response authentication with MD5 or SHA-256
375 * - Supports qop=auth (integrity protection of request)
376 * - Automatic 401 retry with proper response generation
377 * - Handles stale nonce refresh automatically
378 * - NOTE: qop=auth-int (body integrity) is NOT supported
379 *
380 * HTTP_AUTH_BEARER (RFC 6750):
381 * - OAuth 2.0 bearer token authentication
382 * - Token sent in Authorization header
383 * - Application responsible for token management/refresh
384 *
385 * NOT SUPPORTED:
386 * - NTLM (Microsoft proprietary, requires DES/MD4/NTLMv2)
387 * - Negotiate/SPNEGO (requires GSSAPI/Kerberos integration)
388 * - AWS Signature (v2/v4) - use dedicated AWS SDK
389 * - Hawk, HOBA, Mutual, OAuth 1.0a
390 * ============================================================================
391 */

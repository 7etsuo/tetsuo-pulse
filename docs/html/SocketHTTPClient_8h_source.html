335/* ============================================================================
336 * Authentication Types
337 * ============================================================================
338 *
339 * Supported authentication schemes:
340 *
341 * HTTP_AUTH_BASIC (RFC 7617):
342 * - Simple base64(username:password) encoding
343 * - Credentials sent with every request (no challenge-response)
344 * - Only use over HTTPS (credentials transmitted in cleartext)
345 *
346 * HTTP_AUTH_DIGEST (RFC 7616):
347 * - Challenge-response authentication with MD5 or SHA-256
348 * - Supports qop=auth (integrity protection of request)
349 * - Automatic 401 retry with proper response generation
350 * - Handles stale nonce refresh automatically
351 * - NOTE: qop=auth-int (body integrity) is NOT supported
352 *
353 * HTTP_AUTH_BEARER (RFC 6750):
354 * - OAuth 2.0 bearer token authentication
355 * - Token sent in Authorization header
356 * - Application responsible for token management/refresh
357 *
358 * NOT SUPPORTED:
359 * - NTLM (Microsoft proprietary, requires DES/MD4/NTLMv2)
360 * - Negotiate/SPNEGO (requires GSSAPI/Kerberos integration)
361 * - AWS Signature (v2/v4) - use dedicated AWS SDK
362 * - Hawk, HOBA, Mutual, OAuth 1.0a
363 * ============================================================================
364 */
365

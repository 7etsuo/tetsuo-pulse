260/* ============================================================================
261 * Authentication Types
262 * ============================================================================
263 *
264 * Supported authentication schemes:
265 *
266 * HTTP_AUTH_BASIC (RFC 7617):
267 * - Simple base64(username:password) encoding
268 * - Credentials sent with every request (no challenge-response)
269 * - Only use over HTTPS (credentials transmitted in cleartext)
270 *
271 * HTTP_AUTH_DIGEST (RFC 7616):
272 * - Challenge-response authentication with MD5 or SHA-256
273 * - Supports qop=auth (integrity protection of request)
274 * - Automatic 401 retry with proper response generation
275 * - Handles stale nonce refresh automatically
276 * - NOTE: qop=auth-int (body integrity) is NOT supported
277 *
278 * HTTP_AUTH_BEARER (RFC 6750):
279 * - OAuth 2.0 bearer token authentication
280 * - Token sent in Authorization header
281 * - Application responsible for token management/refresh
282 *
283 * NOT SUPPORTED:
284 * - NTLM (Microsoft proprietary, requires DES/MD4/NTLMv2)
285 * - Negotiate/SPNEGO (requires GSSAPI/Kerberos integration)
286 * - AWS Signature (v2/v4) - use dedicated AWS SDK
287 * - Hawk, HOBA, Mutual, OAuth 1.0a
288 * ============================================================================
289 */

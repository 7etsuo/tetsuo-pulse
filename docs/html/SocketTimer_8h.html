Detailed Description

High-performance timer subsystem integrated with the event loop.

Provides high-performance timer functionality integrated with the event loop. Timers are stored in a min-heap for O(log n) insert/delete and O(1) next-timer lookup. Timer callbacks fire automatically during SocketPoll_wait().

Features:

  • One-shot and repeating timers
  • Monotonic clock timestamps (immune to wall-clock changes)
  • Integrated with SocketPoll event loop
  • Thread-safe operations
  • Arena-based memory management

Timing Precision:

  • Uses CLOCK_MONOTONIC for timestamps
  • Millisecond resolution
  • Timers may fire slightly after their deadline due to event loop scheduling

Thread Safety:

  • All operations are thread-safe
  • Callbacks execute in the calling thread (SocketPoll_wait thread)
  • Do not call SocketTimer_cancel() from within timer callbacks

Limitations:

  • Maximum practical timer delay: limited by int64_t range (~292 billion years theoretically) and poll backend timeout (typically ~2.1 billion ms or ~25 days)
  • Timer IDs use 64-bit unsigned integers (wrap around after 2^64 timers, practically unlimited)

Content-Encoding Flags
HTTP Modules » HTTP Client Module

Bit flags indicating supported Content-Encoding methods for HTTP client.

Collaboration diagram for Content-Encoding Flags:

Bit flags indicating supported Content-Encoding methods for HTTP client.

These flags are combined (bitwise OR) in SocketHTTPClient_Config.accept_encoding to specify which encoding methods the client supports in Accept-Encoding header and can decompress in responses.

Default: GZIP | DEFLATE (Brotli optional for smaller payloads).

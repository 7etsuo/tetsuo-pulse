Detailed Description

Secure datagram communication using DTLS over UDP sockets.

Provides DTLS 1.2+ protocol implementation paralleling SocketTLS but optimized for unreliable datagram transport (SocketDgram_T). Enables encrypted, authenticated UDP with forward secrecy, cookie anti-DoS, and full async I/O support via Event System Modules.

Module Relationships

  • Depends on: Core I/O Modules (SocketDgram), Core Foundation Modules (memory/error), Event System Modules (async)
  • Optional: Connection Management Modules (SocketPool for multi-client), Security Modules (SYNProtect integration)
  • Used by: Applications needing secure UDP (IoT, gaming, media streaming)
  • TLS Integration: Shares contexts/certs with SocketTLS where possible

Key Features

  • Non-blocking handshake with poll integration and cookie DoS protection
  • Message boundary preservation (no TCP stream semantics)
  • Configurable MTU, timeouts, ALPN, session resumption
  • Certificate verification, SNI, secure cipher enforcement
  • Cross-platform (OpenSSL/LibreSSL backends)

Requires SOCKET_HAS_TLS=ON and DTLS library support. See CMake options.

Warning
DTLS offers no delivery/reorder guarantees; layer reliability if required. Vulnerable to DoS without cookie exchange; always enable on public servers.

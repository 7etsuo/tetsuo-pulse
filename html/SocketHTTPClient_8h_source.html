253/* ============================================================================
254 * Authentication Types
255 * ============================================================================
256 *
257 * Supported authentication schemes:
258 *
259 * HTTP_AUTH_BASIC (RFC 7617):
260 * - Simple base64(username:password) encoding
261 * - Credentials sent with every request (no challenge-response)
262 * - Only use over HTTPS (credentials transmitted in cleartext)
263 *
264 * HTTP_AUTH_DIGEST (RFC 7616):
265 * - Challenge-response authentication with MD5 or SHA-256
266 * - Supports qop=auth (integrity protection of request)
267 * - Automatic 401 retry with proper response generation
268 * - Handles stale nonce refresh automatically
269 * - NOTE: qop=auth-int (body integrity) is NOT supported
270 *
271 * HTTP_AUTH_BEARER (RFC 6750):
272 * - OAuth 2.0 bearer token authentication
273 * - Token sent in Authorization header
274 * - Application responsible for token management/refresh
275 *
276 * NOT SUPPORTED:
277 * - NTLM (Microsoft proprietary, requires DES/MD4/NTLMv2)
278 * - Negotiate/SPNEGO (requires GSSAPI/Kerberos integration)
279 * - AWS Signature (v2/v4) - use dedicated AWS SDK
280 * - Hawk, HOBA, Mutual, OAuth 1.0a
281 * ============================================================================
282 */
283
